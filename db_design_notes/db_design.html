Data Integrity

We don't want repeating, incorrent values, or broken relationships in and between tables.

  • Entity integrity
  • Referential integrity
  • Domain integrity
{ Entity Integrity }

We should be able to identify a unique entity from a table. For example, in a user table containing only name and phone number as columns, since people may have the same name, we will not be able to idenfity a specific entity from the table sometimes.

{ Referential integrity }

It refers to the accuracy and consistency of data within a relationship.

Referential integrity requires that, whenever a foreign key value is used it must reference a valid, existing primary key in the parent table.

{ Domain integrity }

For example, in table contact, we have a column named phone_number, which must contain 10 digits. When we put some information with invlid format, then it violates domain integrity.

Database Normalization

It is a process structuring a relational database in accordance with a series of so-called normal forms in order to reduce redundancy and improve data integrity.

  • 1st NF
  • 2nd NF
  • 3rd NF

To get 2nd normal form, we need to get 1st normal form, to get a 3rd normal form, get a 2nd normal form first. A 3rd normal form is also a 1st and a second normal form.

{ Second Normal From }

A database is in second normal form if it satisfies the following conditions:

  • It is in first normal form
  • All non-key attributes are fully functional dependent on the primary key

{ Third Normal Form }

Definition

  • The relation R (table) is in second normal form (2NF).
  • Every non-prime attribute of R is non-transitively dependent on every key of R.
